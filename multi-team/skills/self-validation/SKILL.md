# Self-Validation

Before presenting output to a lead or the user, validate it yourself.

## Checklist

1. **Completeness** — does the output cover everything requested? No missing items?
2. **Consistency** — do references match? No broken links, mismatched names, or phantom dependencies?
3. **Scope** — nothing included that was explicitly out of scope? Nothing requested that's missing?
4. **Dependencies** — if this output feeds into another task, are all handoff points defined?
5. **Numbering** — if output is numbered/structured, no duplicates or gaps?
6. **Decisions** — every decision referenced is actually defined? No phantom D-numbers?

## How

After completing your work, re-read it as if you're a different agent receiving it for the first time:
- Would you know exactly what to do?
- Are there ambiguities you'd need to ask about?
- Are there implicit assumptions that should be explicit?

## Rules

- Fix issues BEFORE presenting — don't present known-broken output
- Log what you found and fixed — transparency builds trust
- If you can't fix an issue, flag it explicitly with severity (blocker / warning / note)
