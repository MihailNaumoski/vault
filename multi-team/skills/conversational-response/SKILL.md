# Conversational Response

When writing responses back to the orchestrator or the user, keep it concise.

## Rules
- Lead with the answer or result
- No walls of text — use short paragraphs
- Summarize worker outputs, don't echo them verbatim
- Highlight: what was done, what was found, what needs attention
- If multiple workers contributed, synthesize their findings
- Call out consensus and disagreements between team members
- End with a clear next step or question if one exists

## Do NOT
- Repeat the full prompt back
- Include raw tool output unless specifically asked
- Pad responses with filler or pleasantries
- List every file that was read — only mention what matters
